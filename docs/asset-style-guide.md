# 월드 에셋 스타일 가이드

## 기준
- 레퍼런스: `assets/ui/icons/essence.png`
- 대상: 플레이어, 몬스터, 보스, 보스 약점, 화염구 등 월드에 직접 배치되는 스프라이트
- 배경/UI 에셋은 별도 단계에서 다룬다.

## 파일 규칙
- 포맷은 투명 배경 PNG를 사용한다.
- 기존 에셋은 덮어쓰지 않고 `*_v2.png` 또는 `*_sheet_v2.png` 파일을 추가한다.
- 단일 월드 스프라이트 기준 크기는 256x256이다.
- 플레이어 walk sheet는 기존 프레임 규격인 272x352, 4열 x 2행, 총 8프레임을 유지한다.
- LÖVE 렌더링 필터는 `nearest` 기준으로 확인한다.

## 픽셀 마감 기준
- 실제 렌더링 크기에서 외곽선은 1-2px 두께로 보이게 한다.
- 외곽선은 완전한 검정 대신 짙은 저채도 색을 기본으로 사용한다.
- 내부 명암은 기본색, 어두운 면, 밝은 면, 핵심 하이라이트를 구분한다.
- 팔레트는 한 스프라이트당 12-18색 내외를 목표로 한다.
- 큰 면은 단색 덩어리로 두지 않고 2-3단계 명암으로 덩어리감을 준다.
- 과한 블러, 반투명 연기, 부드러운 그라데이션은 피한다.

## 실루엣 기준
- 128px 이하로 축소해도 플레이어, 일반 몬스터, 희귀 몬스터, 엘리트 몬스터, 보스가 구분되어야 한다.
- 캐릭터의 눈, 뿔, 날개, 입 같은 핵심 특징은 배경과 겹쳐도 읽혀야 한다.
- 보스는 일반 몬스터보다 더 큰 머리/뿔/어깨 실루엣으로 위협감을 준다.
- 화염구는 진행 방향과 중심 코어가 즉시 읽혀야 한다.

## 비교 기준
- `assets/reference/world_sprite_contact_sheet_v2.png`에서 기존 에셋과 v2 에셋을 나란히 비교한다.
- v2가 기존보다 좋아야 하는 항목은 다음 네 가지다.
  - 해상도 밀도
  - 외곽선 일관성
  - 명암 단계
  - 게임 화면에서의 가독성

## 금지
- 기존 파일 직접 덮어쓰기
- 저장 스키마 변경
- 충돌 반경, 대미지, 보상 등 게임 규칙 변경
- 경로 오타를 단일 이미지 fallback으로 조용히 숨기기
